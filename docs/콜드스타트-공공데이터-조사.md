# 콜드스타트: 공공데이터 시딩 조사 (배포 직전 실행용)

> 결론: 가능. 단, 좌표 변환이 필요. 지금은 보류하고 **배포 직전**에 실행한다.

## 데이터 소스
- **LOCALDATA (지방행정인허가데이터개방)**: https://www.localdata.go.kr
- 카테고리: **청소년게임제공업** = 인형뽑기방/오락실. (복합유통게임제공업도 일부 포함)
- 포함 필드: 사업장명, 소재지 주소, 영업상태(영업중/폐업), 인허가일 등
- 공공데이터 = 출처 표기 시 재사용 가능(라이선스 안전). ⚠️ 카카오/네이버 크롤링은 금지.

## ⚠️ 핵심 함정: 좌표계
- 이 데이터의 위치는 **위경도(lat/lng)가 아님**. **중부원점TM(EPSG:5174)** 좌표.
- 앱은 카카오맵 위경도(WGS84, EPSG:4326)를 쓰므로 **좌표 변환 필수**.
- 변환: `proj4` 라이브러리로 EPSG:5174 → EPSG:4326.
  - 또는 주소를 카카오 지오코딩 API로 위경도 변환(주소→좌표)도 대안.

## 실행 절차 (배포 직전)
1. LOCALDATA에서 청소년게임제공업 데이터 다운로드(CSV/API)
2. **영업상태 = 폐업 필터링 제거** (영업중만)
3. 좌표 변환 (TM → 위경도) 또는 주소 지오코딩
4. `places` 테이블에 삽입 (place_name, address, latitude, longitude, created_by=null)
   - created_by=null = "주인 없는" 가게. 유저가 상품 제보하면 그때부터 포인트 발생.
5. 출처 표기: 앱 하단/약관에 "가게 정보 출처: LOCALDATA(행정안전부)" 명시

## 단계 전략
- **1단계**: 한 동네(예: 홍대/강남 한 구역)만 시딩 → "시범 동네" 데모
- **2단계**: 반응 좋으면 전국 확장

## 참고
- 공공데이터포털: https://www.data.go.kr
- 예: 서울시 각 구 청소년게임제공업 인허가 정보 데이터셋 다수 존재
